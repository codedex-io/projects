## Introduction

I miss the days of the internet when people built cool stuff for the sake of making fun things. There were a lot of random sites that everyone visited during my younger years, like [Google Doodles](https://doodles.google/), that embraced creativity and play. 

![Google doodles](https://www.pcworld.com/wp-content/uploads/2025/06/baseball-100900643-orig.jpg?quality=50&strip=all)

Here's a project tutorial that might help bring us back to these days - you'll be building a website where anyone can draw a doodle for a public virtual museum exhibit!

To make this project, we'll be using the classic web dev tools HTML, CSS, and JavaScript, and will be storing our drawings in [Supabase](http://supabase.com/). 

We'll also be using [Vite](https://vite.dev/) to get a local server running our website,  that way you can see your changes immediately. I picked Vite because it handles imports  cleanly, refreshes your website automatically with any changes, and is structured to better handle a project with a realtime database! 

## About Supabase

[Supabase](https://supabase.com/) is a PostgreSQL (Post-Ingres Structured Query Language) development platform. In other words, it's a database system that takes care of everything from setup, deployment, and application scaling - perfect for what we'll be creating today!  

![Supabase](https://nocodestartup.io/wp-content/uploads/2024/09/supabase-02-edited.png)

Whenever a user draws a picture or types out text, we're going to use Supabase to store all that information in real time, and fetch them from a bucket (aka a type of storage container) of our image-text pairs to display drawings in a public gallery!

## Loading your Gallery
When the page first loads, we want to populate the gallery with existing drawings from the database. The `loadGallery()` function reads rows from the drawings table, fetches each drawing's public image URL from the storage bucket, and renders them on the page.

We'll start by writing out the header of our `loadGallery` function and limiting the number of entries to 40. If the gallery element (stored in `galleryEl`) doesn't exist, we stop running the `loadGallery()` function entirely to save time.

```jsx
async function loadGallery(limit = 40, randomize = true) {
  if (!galleryEl) return;
}
```

We open a query on the drawings table (`.from(TABLE)`), select what data we want to reflect in our gallery (in our case, the drawing path, caption, whether or not the drawing is flagged, and a timestamp), order it based on recency (`ascending: false`), and limit the number of drawings fetched to the pool size (which we set as 500 to keep things moving smoothly - if it's less then we don't need to worry about that since 500 is the limit).

Once again, the next few lines will be contained inside the function we just defined, `loadGallery()`.

```jsx
const { data, error } = await supabase
  .from(TABLE)
  .select("path, caption, flagged, created_at")
  .order("created_at", { ascending: false })
  .limit(poolSize);
  
  if (error) throw error;
```

Then, we'll iterate over all the rows we just pulled from the bucket, and add each individual one to the gallery until we hit the maximum number of drawings in `rows`. Since our bucket is public, we'll grab the public URL of the drawing data and add it to our gallery with the `addToGallery()` function, which we'll write shortly after this.

```jsx
 for (const row of rows) {
    const { data: publicData } = supabase.storage.from(BUCKET).getPublicUrl(row.path);
    addToGallery(publicData.publicUrl, row.caption ?? "", !!row.flagged, row.created_at);
 }

}
```

We'll store user inputted text into a variable called title. The `?` symbol is a shorthand version of an `if-else` statement. #### DOM Manipulation

Whenever we have a new drawing to add to our collection of drawings, we'll be generating a `<div>` on the spot for the drawing and caption to live in. This is so we don't have to make many `<div>s` to fill in later, just creating them whenever a user submits a drawing. We're putting each generated `<div>` in a function scoped variable, and assigning the class name “artCard”, which has pre-set CSS properties, to the `<div>`. 

Make sure you're still including this under the `addToGallery()` function!

```jsx
const wrap = document.createElement("div");
wrap.className = "artCard";
```

Then, we'll add tags inside of the element by using `.innerHTML` to change the `<div>`'s HTML content. We'll be encapsulating more `<div>`s inside of the wrap `<div>` we created in the step above, and adding our new drawing, a randomly chosen frame, and a placard (informational cards see in galleries and art exhibits) with information about our drawing, like the month and year. 

If you're curious about what `escapeHtml` does, it makes sure that any characters regularly used in code such as `<>&\`;'?` aren't accidentally interpreted as code - this could mess with your website being able to run at all if users include it in their drawing titles!

```jsx
wrap.innerHTML = `
  <div class="frameStage">
    <div class="artWindow">
      <img src="${publicUrl}" alt="${escapeHtml(title)}" loading="lazy" />
    </div>
    <img class="frameImg" src="${frame.src}" alt="frame" />
  </div>

  <div class="placard">
    <div class="placardTitle">${escapeHtml(title || "Untitled")}</div>
    <div class="placardMeta">Anonymous${monthYear ? ` · ${monthYear}` : ""}</div>
  </div>
`;
```

We'll add ways to make sure that the frame orients properly (horizontally or vertically), and add the finished card to the top of our homepage gallery, with `.prepend(wrap)`.

Supabase will destructure, which means it'll pull out specific values from an object and assign them by finding their keys in the storage bucket and assigning them to `data` and `error`. We add `await` so that the function pauses while your code tries to connect to Supabase online - since network requests can take a bit of time sometimes! 

We'll pull our drawings from our table again - we worked on this earlier in the `loadGallery()` function, and it writes out the same:

```jsx
const { data, error } = await supabase
  .from(TABLE)
  .select("path, caption, flagged, created_at")
  .order("created_at", { ascending: false })
  .limit(poolSize);

if (error) throw error;
```

We want our big gallery to load random drawings, so everyone's drawings have a shot at being seen. Supabase can't shuffle the data it pulls, so we're going to do it ourselves! 

`shuffled` will store the data from our bucket. If the bucket is empty, we'll leave `rows` empty. The template you cloned includes a prewritten `shuffleArray()` function, so we'll shuffle `rows` and take in `data` - aka the drawings. We'll iterate through the first `count` number of our set of shuffled drawings, and store those drawings in the `picked` array by `push()`-ing them in.

`seenPaths` is already declared when you clone the GitHub template, and is used to make sure any data isn't repeated when rendering your gallery.

```jsx
  const shuffled = shuffleArray(data ?? []);
  const picked = [];

  for (const row of shuffled) {
    if (picked.length >= count) break;
    if (!row?.path) continue;
    if (seenPaths.has(row.path)) continue;
    seenPaths.add(row.path);
    picked.push(row);
  }

  return picked;
}
```

### Troubleshooting

- Make sure you've installed all your dependencies - you can do this by going to your root folder in the terminal, and running `npm install`.
- Check that your API keys are accurate and you didn't accidentally remove any characters. Make sure that your variable names are spelled exactly as `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`, and restart your dev server after editing your **.env** file.
- If your drawings aren't submitting to the database, disable RLS in the Supabase dashboard for that table. A bucket not found error means the bucket name in your code doesn't match what you created in Supabase - it should be called "drawings".
- Any other issues? Make sure to inspect element in your browser and check the console for any errors.
