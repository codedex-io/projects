Pretext is a library that is described as a “fast, accurate & comprehensive text measurement & layout” tool created by Cheng Lou. But what does this mean?

When you display text on a webpage, the browser needs to figure out: "How tall will this paragraph be?” and “How many lines does it wrap into?" This would be for example a blog post, or text content for a news article you're reading. 

The traditional way to find out is to literally render the text to the screen (even if it's hidden), then ask the browser to measure it. This is called a DOM measurement, and it's painfully slow, especially if you're doing it for hundreds of items at once.

Pretext solves the problem of calculating the height of a paragraph of line-wrapped text without touching the DOM at all.

## Introduction 

In this project, you'll build a dynamic typography layout using the HTML `<canvas>` element and the `@chenglou/pretext` library to experiment and explore with dynamic text layouts. 

Instead of normal HTML text flow, you'll:

- Render text manually on a canvas
- Make it wrap around floating images
- Reflow dynamically on scroll and resize

<ImageZoom src="https://raw.githubusercontent.com/codedex-io/projects/main/projects/build-a-canvas-typography-layout-with-pretext/images/001.gif" alt="Pretext demo" style={{ width: "100%", height: "auto" }} />

This project is a starting point for you to explore the possibilities of the pretext library and learn

- How to render text with `<canvas>`
- How to detect and avoid overlapping elements
- How to reflow layouts on scroll

## Pretext

`@chenglou/pretext` helps us:

- Measure text
- Break it into lines
- Respect width constraints

We'll use three pretext functions to do just that: `prepareWithSegments()`, `layoutNextLineRange()`, and `materializeLineRange()`.



We'll import the three functions from the library, and set up our `main.js` file: 

```js
import {
  prepareWithSegments,
  layoutNextLineRange,
  materializeLineRange,
} from "@chenglou/pretext";

const FONT = "18px Georgia";
const LINE_HEIGHT = 28;
const CANVAS_WIDTH = 720;
const PADDING = 30;
const TEXT_WIDTH = CANVAS_WIDTH - PADDING * 2;
const IMAGE_GAP = 20;
```

After the setup, create a `const` variable called `ARTICLE`, and in quotes, paste your article or text. If you want some sample text you can use text from this repo [here](https://raw.githubusercontent.com/codedex-io/projects/main/projects/build-a-canvas-typography-layout-with-pretext/sample-article.txt). 

Now, let's connect the JavaScript to the HTML elements we have:

```js
const canvas = document.getElementById("canvas");
const ctx = canvas.getContext("2d");
const container = document.getElementById("container");
const img1El = document.getElementById("img1");
const img2El = document.getElementById("img2");
```

This is so that we are able to draw directly onto our canvas the text and image elements. 

Then, we'll load our article text using `prepareWithSegments()` using the text and the font for the text. This is only run once. 

```js
const prepared = prepareWithSegments(ARTICLE, FONT);
```

### Rendering

Now, we'll start building out the main logic of the project; our render function. 

```js
function render() {
  const dpr = window.devicePixelRatio || 1;
  const containerRect = container.getBoundingClientRect();
```

- `dpr` makes the canvas sharp on high-DPI screens
- `containerRect` helps convert positions correctly


Our next step is to detect our image obstacles, and convert our image positions into canvas coordinates. This will help us treat our images like collisions. In the same `render()` function, add the following:

```js
const obstacles = [img1El, img2El].map((el) => {
  const r = el.getBoundingClientRect();
  return {
    top: r.top - containerRect.top,
    bottom: r.bottom - containerRect.top,
    left: r.left - containerRect.left,
    right: r.right - containerRect.left,
    width: r.width,
  };
});
```

Now we generate each line of text dynamically, and loop until all the text is laid out: 

```js
let cursor = { segmentIndex: 0, graphemeIndex: 0 };
let y = PADDING;
const lines = [];

while (true) {
  let leftCut = PADDING;
  let rightCut = CANVAS_WIDTH - PADDING;
```

We track the left and right edges of the available text band separately so multiple images on the same line can each tighten one side without overwriting each other. We'll then check if each line overlaps an image.

```js
  for (const obs of obstacles) {
    const lineTop = y;
    const lineBottom = y + LINE_HEIGHT;

    if (lineBottom > obs.top && lineTop < obs.bottom) {
      const obsCenter = (obs.left + obs.right) / 2;
      const canvasCenter = CANVAS_WIDTH / 2;
```

For the following lines of code, we're asking to detect if the image is on the right to then tighten the right edge, and if the image is on the left to push the left edge inward. Depending on your desired layout or behaviour you want to execute, this could change. 

```js
      if (obsCenter > canvasCenter) {
        // Image on right → tighten the right edge
        rightCut = Math.min(rightCut, obs.left - IMAGE_GAP);
      } else {
        // Image on left → push the left edge inward
        leftCut = Math.max(leftCut, obs.right + IMAGE_GAP);
      }
    }
  }

  const xOffset = leftCut;
  const lineWidth = Math.max(50, rightCut - leftCut);
```

Now, we'll ask pretext for the next line that fits using `layoutNextLineRange()` and `materializeLineRange()` to then store it. 


```js
  const range = layoutNextLineRange(prepared, cursor, lineWidth);
  if (range === null) break;

  const line = materializeLineRange(prepared, range);

  lines.push({ text: line.text, x: xOffset, y });

  cursor = range.end;
  y += LINE_HEIGHT;
}
```

After the layout is adjusted, we'll resize the canvas to fit all the text: 
```js
 const totalHeight = y + PADDING;

  canvas.width = CANVAS_WIDTH * dpr;
  canvas.height = totalHeight * dpr;

  canvas.style.width = CANVAS_WIDTH + "px";
  canvas.style.height = totalHeight + "px";

  ctx.scale(dpr, dpr);
```

We'll then draw the background and the text. This can be your creative choice. 

```js
  ctx.fillStyle = "#fffdf8";
  ctx.fillRect(0, 0, CANVAS_WIDTH, totalHeight);

  ctx.font = FONT;
  ctx.fillStyle = "#2a2a2a";
  ctx.textBaseline = "top";

  for (const line of lines) {
    ctx.fillText(line.text, line.x, line.y + 4);
  }
}
```
Note that this is the end of the `render()` function.

### Re-rendering on Scroll and Resize

When the user scrolls or resizes the window, we re-run `render()` so pretext can fit the text to the new constraints. We'll use `window.addEventListener()` for this: 

```js
let ticking = false;

window.addEventListener("scroll", () => {
  if (!ticking) {
    requestAnimationFrame(() => {
      render();
      ticking = false;
    });
    ticking = true;
  }
});
```

Finally, add the last two lines of code to re-render based on size, and call the `render()` function to display everything. 
```js
window.addEventListener("resize", () => render());
render();
```

## Drag the Images

Scrolling already reflows the text, but you can't really feel how fast pretext is until you drag an image around. Grab an image, move it, and every line rewraps as it moves.

The HTML and CSS we set up earlier already cover the visual side of this. The images are absolutely positioned inside `#container`, `cursor: grab` invites the user to grab them, and `touch-action: none` tells the browser not to interpret a touch on the image as a scroll. The rest is JavaScript.

Add this `makeDraggable` helper to **main.js**, then call it on each image:

```js
function makeDraggable(el) {
  let dragging = false;
  let curX = null;
  let curY = null;

  el.addEventListener("pointerdown", (e) => {
    if (curX === null) {
      const cs = getComputedStyle(el);
      curX = parseFloat(cs.left);
      curY = parseFloat(cs.top);
    }
    dragging = true;
    el.setPointerCapture(e.pointerId);
  });

  el.addEventListener("pointermove", (e) => {
    if (!dragging) return;
    curX += e.movementX;
    curY += e.movementY;
    el.style.left = curX + "px";
    el.style.top = curY + "px";
    render();
  });

  el.addEventListener("pointerup", (e) => {
    dragging = false;
    el.releasePointerCapture(e.pointerId);
  });
}

makeDraggable(img1El);
makeDraggable(img2El);
```

A few of the choices inside this helper are worth pulling apart.

- `pointerdown`, `pointermove`, and `pointerup` are the unified events that fire for mouse, touch, and stylus, so the same handlers work on every device without any branching. The `touch-action: none` rule in CSS is the partner setting. It stops the browser from interpreting a touch on the image as a scroll gesture and swallowing our `pointerdown`.

- The `draggable="false"` attribute on each image is doing its own quiet work. Without it, the browser starts its built-in image drag-and-drop the moment you press, which eats every `pointermove` and `pointerup` that should reach our handler. The drag would feel broken in two ways at once: a ghost image would float under the cursor, and our `dragging` flag would stay stuck at `true` after the press because no `pointerup` ever arrives.

- The `dragging` flag exists because `pointermove` fires whenever the pointer is over the image, even when no button is pressed. The flag gates the handler so it only does work between a matched `pointerdown` and `pointerup`. Without it, the image would jump every time the cursor passed over it.

- For position, we read the CSS value once and cache it. The first time you grab an image, it has no inline `style.left` yet, only the position from the CSS rules. We read that with `getComputedStyle`, parse the pixel string into a number, and store it in `curX` and `curY`. From then on, we maintain our own state and write back to `style.left` and `style.top`. We avoid re-reading the DOM on every move because calls like `getBoundingClientRect` force the browser to flush pending style and layout work, which makes drags feel sticky. Reading a cached JavaScript number is free by comparison.

- `setPointerCapture` is what keeps the drag alive when the cursor moves faster than the image can follow. Without it, the pointer leaves the image's bounding box and pointer events stop firing on the element; the drag silently dies. Calling `setPointerCapture(pointerId)` tells the browser to route every event from that pointer to the image until `pointerup`, when we hand it back with `releasePointerCapture`.

- Using `e.movementX` and `e.movementY` rather than absolute coordinates is the choice that matters most here. The tempting approach is to compute the new position from `e.clientX` and the container's bounding rect, but that breaks the moment anything throws the math off, like a scrolled page or a CSS transform on an ancestor. `movementX` and `movementY` report how far the pointer moved since the last event, in raw pixels. We add the delta to the cached position, write it back, and the image moves the same distance the cursor did. There is no coordinate translation to get wrong.

The `render()` call inside `pointermove` is what reflows the text. On every move event, the layout gets rebuilt from scratch: pretext re-walks the article, the obstacle code re-reads the image's new `getBoundingClientRect`, and the canvas gets repainted. Because pretext never measures text through the DOM, doing this per frame stays fast enough to feel instant.

Refresh and drag an image. The text rewraps in real time.

## Bonus: Create an Image Spinner

You're techncially done with the project! But you can add some extra flair by making the images spin! 

<ImageZoom src="https://raw.githubusercontent.com/codedex-io/projects/main/projects/build-a-canvas-typography-layout-with-pretext/images/007.gif" alt="Image spinner demo" style={{ width: "100%", height: "auto" }} />
In **main.js**, you'll need to add a `addSpinFadeOnClick()` function to the `img1El` and `img2El` elements. This function will add a `spin` animation to the images when they are clicked.

```js
function addSpinFadeOnClick(imgEl) {
  if (!imgEl) return;
  imgEl.style.pointerEvents = "auto";

  imgEl.addEventListener("click", () => {
    imgEl.animate(
      [
        { transform: "rotate(0deg)" },
        { transform: "rotate(1440deg)" },
      ],
      {
        duration: 5000,
        easing: "ease-out",
        fill: "forwards",
      }
    );
  });
}

addSpinFadeOnClick(img1El);
addSpinFadeOnClick(img2El);
```

This will add a `spin` animation to the images when they are clicked. The animation will start at 0 degrees and end at 1440 degrees, and will take 5 seconds to complete. The easing is set to `ease-out`, which means the animation will start slow, speed up, and then slow down.
